# 핵심가치 정의

(1) 지적재산권 중 단 기간내 성장곡선을 그리는 자산은? 영화대본, 웹툰스토리, 드라마대본 즉, 네러티브가 있는 스토리임. 

(2) 스토리체인은 기존의 스토리 콘텐츠 사업자들이 더욱 가치있게 스토리 자산으로 사업을 할 수 있도록 하는 블록체인 기반  '프로토콜' (계약킷, 토큰, 개발툴) 을 제공함. 

(3) 토큰의 가치 설계 방향

-  많은 프로덕션, 플랫폼사들이 스토리체인 프로토콜인 storiOS 스토리오에스를 사용 할 때, 토큰의 회전율이 올라가는 구조
-  많은 창작자, 독자, PD 들이 스토리체인 dApp 을 사용 하며 아이템 소진, 스토리성장 그래프를 만들 때 생태계의 GDP가 올라 가는 구조

# 토큰 이코노미 특징 
"이익 공유 경제 모델 " 

* Augur와 유사한 "Work Token Model"
* 스토리프로듀서가 스토리자산에 Staking, Staking 금액과 스토리당 성장그래프 합산한 기준으로 토큰 추가발행(Mint)
* Mint한 량의 80%는 유저 보상풀인 Eco-Pool 로 비축, 20% 주별 성장그래프 달성한 스토리에 리워드
* 초기 유입을위해 Eco-Pool 에서 유저에게 많이 보상하다가 점점 줄어 들고, 시간이 지날 수록 mint 된 량에서부터 유저보상 하는 량을 높임. 


작가, 독자, 피지 3자 참여자들이 스토리를 성장 시키는 정도에 따라 
미래 수익 분배율 (기여도)가 달라짐. 

작가: 창작과 재미에 대한 책임으로 기여도를 받음
독자: 리액션을 하고 기여도를 받음
피디: 서포팅(스테이킹)을하고 스토리 검증권한을 지니게 됨. 검증을 하면 기여도를 받음 

각 3자의 노력과 보상을 성장지표 기반으로 기여도를 산출 해 내는 것이 핵심 로직

# 왜 블록체인 기술이 필요한가
* 스토리 창작에서 작가의 창작기여도는 미래 판권 판매 수익을 배분할 데이터이기에 위변조 없이 저장 되어야 함. 
* 신뢰비용을 월등히 낮추며 새로군 가치 시장 창출 가능 
    * 대본 거래 시장은 '아이템 비밀주의'임 오프라인에서 거래 하려면 신뢰비용이 높음
    * 블록체인의 특정 중 '비대면 신뢰 계약' 의 특성을 응용하면 이를 해결 가능 함. 
    * 제작사가 제작을 못하고 유휴적채한 대본은 제작완료 편수의 7배 정도임. 이를 리세일 리세일하여 유동화 할 방안이 없음.
* 스토리를 '자산' 화 할 수 있음
    * 스토리 (대본, 시나리오, 소설, 희곡, 게임시나리오) 등은 2~3차 재제작을 염두 한 IP 자산임
    * 2~3차 저작물이 나올 때마다 가치 상승이 있음. 가치 상승을 신뢰 있게 하고 미래 수익을 지급 보증하려면 스마트컨트랙트 를 사용 해야 함.

# 우리는 왜 이 프로젝트를 하게 되었는가? 
    1. 아이덴티적 : 영화 창작을 해 오며 몸소 문제를 느껴 왔기에
    2. 산업적으로 : 영화 산업의 미들맨의 초과 수익이 작가와 관객들에게 떨어지지 않음을 느끼고
    3. 기술적으로 : 공동 창작에 대한 협의 알고리즘을 획기적으로 만들고 싶어서  

# 어떤 문제를 해결하려 하는가? 
    1. 불공정 2차 판권 계약으로 헝그리한 원작 작가들
    2. 재미가 불확실한 작가에 대한 리스크를 (집필료 등)  혼자 감당하는 제작사들
    3. 유휴적채된 스토리 판권을 리세일 할 판로를 못 찾는 제작사들
    4. 정량적 데이터 없는 시나리오를 기반으로 펀딩을 받으러 다니는 제작사들
    5. 작가-제작사간의 집필료 지불 연기 문제 (DB)
    6. 협업 작가간 수익 배분 문제
    7. 협업 창작시 위변조 없는 기여도 저장
    8. 일반인이 영화/드라마등의 초기 시나리오 개발 단계에서 투자 할 수 없었음. (미들맨들이 룸을 열어주지 않음) (일반인에게 열어주면 정산의 투명성, 복잡성 문제)

# Why 블록체인? 
    1. 블록체인을 쓰는 이유는? 
        1. 비대면 신뢰거래 : 닷컴시대, 모바일 시대가 이룰 수 없었던 대본 거래를 비대면 신뢰 거래 할 수 있는 점
        2. 창작기여도의 공증 효과 
        3. 대본을 투자 대상 상품으로 ASSET화 Tokenize  할 수 있는 점
        4. 지금은 데이터 노동자의 시대이다. 
        - 유일하게 노동의 댓가에 대해 정당한 대우를 받지 못하고 있는 것이 창작 노동 데이터이다.모두 창작의 결과를 갖고 평가 한다. 
        - 우리는 결과가 아닌 노동 과정의 데이터 생산에 대해서도 보상을 주려 한다. 
        5. 일반인이 여유자금으로 문화콘텐츠인 대본에 정산의 투명성을 보장받으며 투자할 기회를 제공해 준다. (스토리를 기반으로 한 펀드상품) 
   
   2. 무엇을 온-체인 할 것인가? 
        1. 기여도 데이터를 저장 한다.
            1. 기여도 데이터는 유저군별 행동을 기준으로 3종이 있음 : (1)창작기여도, (2)리액션기여도, (2)서포트기여도 
            2. 기여도 데이터는 시간별 3종이 있음 : (1)잠재기여도, (2)주별확정기여도, (3)누적기여도
        2. 스토리원문은 블록체인에 저장하지 않는다. 
        - 공증효과를 위해 원문의 해쉬값은 일정 주기별로 저장한다. 
        - 스토리 원문 텍스트는 IPFS와 중앙서버 DB에 저장
    2. 블록체인의 한계 
    - 블록체인은 기본적으로 데이터 스트럭쳐, 토큰이코노미를 통한 행동경제, 컨센서스로 봄. 
    - 블록체인이 모든 문제를 해결 할 수 없다. 어플리케이션 레이어에서 블록체인 기반 데이터를 응용해서 해결 가능 하다. 
    - 따라서 기술적 탈중앙화는 일부분만 가능하며 생태계의 철학적 탈중앙화는 상당부분 어플리케이션을 사용하는 3자군 유저의 협의 과정에서 일어난다. 

# 어떤 문제를 블록체인으로 해결 하려 하는 가?
    1. 우리가 블록체인으로 보는 시각은 크게 3가지이다. 
    - 데이터베이스 측면 : 비가역적 공유 데이터로써의 블록체인 
    - 크립토커런시 측면 : 유저의 동기부여 요인을 주는 크립토커런시 적 블록체인. 행동경제학 
    - 컨센서스 : 사회민주적 컨센서스 방법으로써의 블록체인
    2. 위와 같은 시각으로 스토리체인이 응용 해결 하는 문제 
    - 데이터베이스 : 다수가 하나의 스토리에 창작 기여시, 기여도를 위변조 없이 저장 하는 것. 일반인들이 정산의 투명성을 신뢰하며 스토리에 투자 가능 함. 
    - 프로토콜 비지니스 : 스토리체인 영리사업자가 사라져도, 써드파티 사업자들이 지속적으로 인류의 공용데이터로써의 스토리를 서비스 할 수 있게하려는 것. 
    - 크립토커런시: 작가, 독자, 피디및 일반인 의 행동경제학을 만들려한다. 
    - 컨센서스 : PoP 스토리에 참여하는 3자 고객군이 스토리의 최소 완성도를 서로 합의함.
    이야기의 최소 재미를 정량화해서 시각적으로 보여주는 것이 스토리성장지수이다. 이것을 만드는 3자 참여자들이 각자의 책임행동을 통해 합의를 이뤄낸다. 
    3자가 공동의 목표인 스토리성장지수를 키우려 열심히 노력하지 않으면 보상 받기 힘든 구조, 이를 통해 함께 스토리의 퀄리티 합의를 주별로 이룬다. 
    - 탈중앙화의 범위: 재화를 만드는 스토리 노동자의 노동 수치, 즉 기여도와 기여도 산정 로직을 스마트컨트랙트에서 저장하려한다. 


# 이 프로젝트의 장점
   1. 비교적 오라클(Oracle) 문제 없음: 오프라인 활동 결과 데이터와 연계하는 데이터 신뢰 문제가 없음. 
   2. 잠재성장률이 매우큼. 시장규모, 성장률 대비 주목받지 못하는 비지니스 영역(니치시장) 
         - 국내 웹소설, 매년 2배 성장률 
             - 동영상 미디어 시장이 커지면 그 동영상을 만들기 위한 스크립트(대본) 시장도 비례해서 커짐
               -> 비유적으로 전기자동차 시장 성장이 예측된다면, 전기배터리 시장에도 투자함. 마찬가지로 동영상미디어 시장이 커짐에 따라 시간을 디자인 해야 하는 기승전결이 필요한 스토리 창작과 IP 거래 시간이 중요함
 
  3. 스토리 산업이 본격적으로 디지털 플랫폼화 되어 있으나 비대면신뢰거래를 할 수 없어 활성화 되지 못함. (공증을 온라인으로 쉽게 받을 수 없었기 때문) 
  4. 스토리를 토크나이즈, 일반인도 쉽게 투자 할 수 있게 한다. 일반인은 은행, 주식에 투자하려고 해도 수익율이 낮다. 
  일반인에게 장벽이 있었던 영화화 되기전 스토리에 투자할 기회를 제공. 
  5. 일반인 투자자의 엑시트 기간이 빠른 지적재산권 투자상품.

# 토큰의 수혜자가 누구인가? 
* 스토리 성장에 기여하는 일반인 누구나 (토큰 홀더) 

# 토큰의 모델 패턴은? 
* 워크 토큰 + 이익공유 경제 모델형 
* 워크 토큰
- 토큰을 스테이킹하면 재화를 성장 시킬 권리 확보
- 공동으로 재화를 성장 시켜야만 나누어 가질 수익율이 커지는 토큰  

# 토큰 내재 가치 
  1. 스토리에 기여한 기여도(Contribution Rate) 및 스토리 성장지수(Growth Index) 를 기준으로 
  한 스토리의 최소 적정가 펀더멘털을 거래시 거래자간 합의에 이를 수 있음. 
  2. 한 스토리를 기준으로 2차 판권으로 영화/출판화 될시, 어느 Growth Index 에 오른 스토리가 얼만큼의 판권 가격으로 영화, 출판화 되었는지 데이터를 모아 토큰의 부가가치를 만들어내는 성장률을 추정 할 수 있음. 
  3. 인플레이션 발생을 스토리 실질생산량과 퀄리티를 기준으로 적절하게 발생시킴 
  4. 이익공유경제 모델 Profit Share Economy Model 
    * 참여가자 함께 키운 재화(=스토리)에서 발생한 이익을 공유하려면 오직 최소 DAPP 에서 써야만 하는 TORI가 필요함 
    * 포인트, 화폐, 레퓨테이션, 투표권까지 모두 TORI 로 치환 되어 생태계내 활동(소진/획득) 할 때 마다 TORI의 회전율 발생하는 구조. 
  5. 스토리를 확대하는 포크, 브랜치를 하려면 최소 TORI 가 필요함 

# 토큰 회전율(Velocity)
* 크게 3군데의 용처 
    * 퀄리티있는 스토리의 양
    * 댑 서비스가 판매할 디지털 아이템(생태계 매거니즘 안에 책임과 보상으로 만나는 디지털 아이템 들)
    * 프로토콜 사용하는 댑의 수

# 토큰의 희소성 (Scarcity)
* PD 군 유저가 스토리에 Staking 행위(서포트) 
* 스테이킹하는 양만큼 Lock 되는 효과로 희소성을 만들어 냄.
